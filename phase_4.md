#Этап 4: Слой поселков и сложная бизнес-логика
*Цель:* Привязать данные к физическим объектам (CRM).
-Поселки: Загрузить данные из bx24_location.
-Пространственный Join: Сгруппировать абонентов по ad (идентификатор поселка из MySQL).
-Сводные метрики по поселку:
    Всего домов (потенциал).
    Активных (из point_customer_active).
    Заблокированных (из point_customer_sysblock).
    Визуализация: Добавить маркеры поселков, при клике на которые открывается Popup с мини-таблицей показателей.

*Результат:* Инструмент для работы региональных менеджеров.
