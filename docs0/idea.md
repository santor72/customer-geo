# Идея 
idea.md
карта с расположением клиентов и данными о них с выводом данных по месяцам
Слои - абоненты с кластеризацией, поселки(с данными по абонентам в них), гексагоны от Uber
## Схема данные

*point_utm5.acct_addr_coordinates* - clickhouse, таблица абонентов
account_id - идентификатор абонента
lat - широта
lng - долгота
ad - идентификатор поселка

*point_add.bx24_location* - clickhouse, таблица поселков
id - идентификатор поселка
title - название поселка
ufCrm11_1732803360 - широта
ufCrm11_1732783301 - долгота

*point_add.point_customer_active* - clickhouse, таблица  активных абонентов на каждый месяц
account_id  - идентификатор абонента
recv_mon - последний день месяца для которого сформирован отчет

*point_add.point_customer_sysblock* - clickhouse, таблица заблокированных  абонентов
account_id  - идентификатор абонента
recv_mon - последний день месяца для которого сформирован отчет

*dtall* - clickhouse, таблица списаний 
account_id  - идентификатор абонента
discount - сумма списания
discount_date - дата списания, unix timestamp в секундах, целое

*pt* - clickhouse, таблица платежей
account_id  - идентификатор абонента
payment_absolute - сумма платежа
actual_date - дата платежа , unix timestamp в секундах, целое
method - етод платежа, целое, интересуют [6553998,103,104]

данные выводятся в разрезе месяца на который строится карта

## Соглашение о технологиях
основной язык backend - python
Для работы с данными - pandas, h3, geopandas
Для создания карты - Kepler.gl 
Для развертывания - flask


## Соглашения по интерфейсу
### Общее
*Слой гексагоны* - H3 гексагоны с данными - активные абоненты/заблокированные абоненты/количество списаний-сумма списаний/количество платежей-сумма платежей
*поселки* - точки отражающие поселки, всплывающий baloon с данными по абонентам в поселке - активные абоненты/заблокированные абоненты/количество списаний-сумма списаний/количество платежей-сумма платежей
*абоненты* - ScatterplotLayer

### Логика переключения (Zoom Trigger)
*Zoom 1-10* (Обзор области/города): Видны крупные Гексагоны. Они показывают «тепловую карту» — где больше денег. слой поселков с их статистикой
*Zoom 11-14* (Район/Поселок): Появляются границы Поселков (полигоны) с их статистикой.
*Zoom 15+* (Улица/Дом): Гексагоны исчезают (или становятся прозрачными), и проявляются ScatterplotLayer — маленькие точки, каждая из которых — конкретный абонент.

## Архитектура 
два независимых контура: 
    Backend (API) — занимается данными, считает метрики и отдает JSON. 
    Frontend (Visualization) — занимается отрисовкой, переключением слоев и кэшированием. 

### Backend (python)
Сервер не генерирует HTML. Он отдает "сырые" данные в формате GeoJSON или JSON. 

Структура эндпоинтов: 

    GET /api/overview (Данные для Зума 1-10) 
         Вход: month (например, 2023-12-01).
         Логика:
             Запрос к CH: агрегация платежей/списаний по координатам.
             Преобразование координат в H3 индексы (разрешение, например, 4 или 5).
             Группировка данных внутри гексагонов.
             
         Выход: GeoJSON FeatureCollection, где каждый Feature — это гексагон с свойствами (сумма платежей, кол-во активных и т.д.).
         Дополнительно: Отдает данные по поселкам (агрегация по полю ad).
          

    GET /api/points (Данные для Зума 11+) 
         Вход: month, bbox (координаты видимой области карты), zoom.
         Логика:
             Фильтрация абонентов, попадающих в bbox.
             Если зум < 15: можно вернуть "прореженные" данные или просто точки без детальной инфо, чтобы облегчить вес.
             Если зум >= 15: вернуть полный набор данных для видимой области.
             
         Выход: JSON список объектов [{id, lat, lng, type, sum_pay}, ...].
          
### Frontend (JavaScript + Kepler.gl) 

Используем Kepler.gl (через CDN или npm). Это библиотека, специально созданная для визуализации больших данных на картах. 

Логика на JavaScript: 

    Инициализация карты: Используем MapView (Kepler.gl) поверх Mapbox или Google Maps API. 
    Обработчик изменения вида (.onViewStateChange):
         Отслеживаем текущий zoom.
         Если zoom < 11: Показываем слой H3HexagonLayer. Прячем/очищаем слой точек.
         Если zoom >= 11: Прячем слой H3HexagonLayer. Запускаем загрузку точек через fetch('/api/points?bbox=...').
         Если zoom >= 15: Меняем стиль отрисовки точек (делаем их крупнее, добавляем Popup при клике).
          
    Кэширование: Сохраняем загруженные данные в JS-переменную при смене месяца, чтобы не грузить повторно при перемещении карты. 

### Интерфейс и деплой 

    HTML-шаблон:
         Слева/сверху: панель управления (выбор месяца, чекбоксы "показать заблокированных").
         На фоне: <div id="map-container"></div>.
         Справа: легенда (цвет = сумма платежей).
          
    Flask:
         Раздает статику (HTML/JS/CSS).
         Проксирует запросы к API.
