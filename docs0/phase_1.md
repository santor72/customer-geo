Шаг 0 — Прототип (быстрый proof‑of‑value)

## Цель
Показать карту с переключением слоев по zoom с данными за текущий месяц на минимальном объеме работ.

## Результат шага
- Frontend отображает карту и подгружает данные из Backend.
- Backend отдает 3 слоя (H3/поселки/абоненты) хотя бы в упрощенном виде.
- Данные могут считаться «на лету» (допускаются тяжелые запросы и pandas), но с ограничениями bbox

## Работы
- Настройка соединений: Написать модули подключения к MySQL (через pymysql или sqlalchemy) и ClickHouse (clickhouse-connect).
- Сбор данных: Написать функцию, которая делает JOIN таблицы координат (acct_addr_coordinates) и активных абонентов (point_customer_active) за последний доступный месяц.
- Создать простую картучтобы отобразить данные абонентов.
- создать необходимые API 
- Результат:  видите «облако» своих клиентов на карте в браузере.
## API
- FastAPI
- Ручки:
  - `GET /api/v1/months`
  - `GET /api/v1/layers/settlements?month=current&bbox`
  - `GET /api/v1/layers/h3?month=current&bbox&h3_res`
  - `GET /api/v1/layers/month=current?month&bbox&limit`

### 3) Frontend (минимальный)
- kepler.gl или deck.gl
- UI:
  - отображение слоя абонентов с кластеризацией
  - Триггер запросов по событиям карты (moveend/zoomend) с debounce.
## Критерии приемки
- За 5–10 секунд можно увидеть карту для текущего месяца.
- Ошибки параметров корректно валидируются.
