# Шаг 0 — Прототип (быстрый proof‑of‑value)

## Цель
Показать карту с переключением слоев по zoom и выбором месяца на минимальном объеме работ.

## Результат шага
- Frontend отображает карту и подгружает данные из Backend.
- Backend отдает 3 слоя (H3/поселки/абоненты) хотя бы в упрощенном виде.
- Данные могут считаться «на лету» (допускаются тяжелые запросы и pandas), но с ограничениями bbox.

## Работы
### 1) Договориться о семантике месяца
- Ввести входной параметр `month=YYYY-MM`.
- Внутри приводить к `recv_mon = last_day(month)`.
- Зафиксировать: платежи/списания считаются по транзакциям внутри календарного месяца.

### 2) Backend (минимальный)
- FastAPI/Flask
- Ручки:
  - `GET /api/v1/months`
  - `GET /api/v1/layers/settlements?month&bbox`
  - `GET /api/v1/layers/h3?month&bbox&h3_res`
  - `GET /api/v1/layers/subscribers?month&bbox&limit`

Ограничения:
- Ограничить bbox на `subscribers` (например max площадь/диагональ).
- `limit` обязателен.

### 3) Frontend (минимальный)
- kepler.gl или deck.gl
- UI:
  - выбор месяца
  - выбор метрики
  - отображение слоя согласно zoom
- Триггер запросов по событиям карты (moveend/zoomend) с debounce.

### 4) Мониторинг (минимальный)
- Логировать все запросы: endpoint, bbox, время, размер ответа.

## Критерии приемки
- Переключение слоев по zoom работает.
- За 5–10 секунд можно увидеть карту для выбранного месяца.
- Ошибки параметров корректно валидируются.

## Риски/долги
- Производительность ограничена; нужны витрины и кэш на следующем шаге.
