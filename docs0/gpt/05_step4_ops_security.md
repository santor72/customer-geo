# Шаг 4 — Продакшенизация (deploy, безопасность, SLO)

## Цель
Подготовить систему к эксплуатации: надежный деплой, доступы, SLO, бэкапы, алерты.

## Результат шага
- Описан deploy (Docker/K8s), секреты, роли CH.
- Настроены алерты и дашборды.
- Определены SLO и процедуры восстановления.

## Работы
### 1) Деплой
- Backend + Redis + Frontend.
- Конфигурация через env/secret.
- Healthchecks:
  - `/healthz`
  - `/readyz` (проверка CH/Redis)

### 2) Доступы и секреты
- Отдельные учетные записи CH:
  - `etl_writer`
  - `api_reader`
- Секреты в vault/secret manager.

### 3) SLO/метрики
- SLO (пример):
  - H3/settlements p95 < 1s
  - subscribers p95 < 2s при bbox <= X
- Дашборды:
  - latency, rps, error rate
  - cache hit rate
  - CH query time

### 4) Алерты
- 5xx rate
- p95 latency
- Redis down / CH down
- рост размера ответов

### 5) Бэкапы и восстановление
- Политика бэкапов для CH (если витрины строятся заново — можно не бэкапить их, но нужно понимать сроки rebuild).
- Документированный runbook: rebuild месяца M, очистка кэша, rollback версии.

## Критерии приемки
- Сервис переживает рестарт без ручных действий.
- Есть наблюдаемость и алерты.
- Есть documented runbook.
