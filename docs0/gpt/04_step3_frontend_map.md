# Шаг 3 — Frontend карты (UX, производительность, слои)

## Цель
Сделать удобный интерфейс для аналитики по месяцам и метрикам, обеспечить плавную работу при перемещении карты.

## Результат шага
- UI: выбор месяца, метрики, базовые фильтры.
- Слои корректно переключаются по zoom.
- Запросы делаются по bbox с debounce и клиентским кэшем.

## Работы
### 1) Управление состоянием
- Глобальный state: `month`, `metric`, `filters`, `data_version`.
- При смене месяца:
  - запрос `/version` → обновить `data_version`
  - сбросить кеш слоев

### 2) Логика слоев по zoom
- zoom 1–10: `h3 + settlements`
- zoom 11–14: `settlements` (+ optional h3 higher res)
- zoom 15+: `subscribers + settlements`

### 3) Запросы по bbox
- `moveend/zoomend` → запрос слоев.
- Debounce 200–400ms.
- Ограничение: не запрашивать subscribers при слишком большом bbox.

### 4) Визуализация метрик
- Цвет/размер по `metric`.
- Tooltip:
  - значения метрик
  - производные (block_rate, net, arpu)

### 5) Экспорт (опционально)
- Выгрузка таблицы по выбранным фичам (гексы/поселки) через отдельную ручку или прямо из данных.

## Критерии приемки
- Перемещение карты не вызывает «шторм» запросов.
- На обычном ноутбуке интерфейс остается отзывчивым.
- Слои включаются/выключаются предсказуемо.

## Риски/долги
- Если GeoJSON станет тяжелым: перейти на MVT и рендер векторных тайлов.
