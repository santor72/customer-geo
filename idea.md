# Идея 
idea.md
карта с расположением клиентов и данными о них с выводом данных по месяцам
Слои - абоненты с кластеризацией, поселки(с данными по абонентам в них), гексагоны от Uber

### Что пользователь делает
1. Выбирает **месяц** (формат `YYYY-MM`).
2. Открывает карту и видит агрегаты по территории.
3. При приближении (zoom) слой «детализируется»:
   - 1–10: гексы (H3) + поселки
   - 11–14: поселки (и при наличии — полигоны поселков)
   - 15+: абоненты (точки) + поселки
4. Меняет показатель (например, `payments_sum`, `charges_sum`, `active_cnt`, доля блокировок), применяет фильтры.

## Данные
данные выводятся в разрезе месяца на который строится карта
месяц либо 
* current - текущий и данные берутся из таблиц витрины текущего месяца
* YYYY-MMM-DD - данные берутся из историческиз витрин

### Соглашение о модели месяц
В таблице это `recv_mon = последний день месяца`
* в API передаём либо `month=current`(для текущего месяца) либо `month=YYYY-MM`(для запроса по предыдущим месяцам)
* все витрины строятся по `recv_mon`, чтобы не было расхождений.

### Схема данных

Витрины mysql для текущего месяца в файле current.sql
Витрины clickhouse для истории по месяцам в файле month.sql

### Логика переключения (Zoom Trigger)

*Zoom 1-10* (Обзор области/города): Видны крупные Гексагоны. Они показывают «тепловую карту» — где больше денег. слой поселков с их статистикой

*Zoom 15+* (Улица/Дом): Гексагоны исчезают (или становятся прозрачными), и проявляются ScatterplotLayer — маленькие точки, каждая из которых — конкретный абонент.

## Контракты API (минимально достаточные)

### Справочник периодов

* `GET /api/v1/months` → список доступных `YYYY-MM` (или `recv_mon`)

### Гексагоны 

* `GET /api/v1/layers/h3`

  * params: `month=[YYYY-MM|current]`, `bbox=minLng,minLat,maxLng,maxLat`, `h3_res=...`
  * response: GeoJSON FeatureCollection (Polygon) или компактный JSON `{h3_index: metrics}`

**Важно:** резолюцию H3 можно выбирать автоматически:

* zoom 1–10 → res 6–7
* zoom 11–14 → res 8–9
  (точные значения подберёте под вашу территорию)

### Посёлки 

* `GET /api/v1/layers/settlements`

  * params: `month=[YYYY-MM|current]`, `bbox`
  * response: GeoJSON Point + properties(metrics)

### Абоненты 

* `GET /api/v1/layers/subscribers`

  * params: `month=[YYYY-MM|current]`, `bbox`, `limit` (и пагинация/курсор)
  * response: точки (GeoJSON или массив)

## Логика фронта 


* **Zoom 1–10:**

  * слой H3 “heat” (метрика выбирается: payments_sum / charges_sum / net)
  * слой поселков как точки (чтобы можно было быстро кликать)
  * абоненты: точки по bbox c кластеризацией
  
* **Zoom 11–14:**

  * H3 на более высокой res *или* выключить и оставить поселки+границы
  * поселки: точки + (опционально) полигоны поселков, если появятся
  * абоненты: точки по bbox c кластеризацией

* **Zoom 15+:**

  * поселки остаются (легкий слой)
  * абоненты: точки по bbox c кластеризацией

* Иструмент для включения отлючения слоев
